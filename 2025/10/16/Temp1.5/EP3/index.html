【Temperature 1.5 的日常】EP3: LangChain - Tools, 賦予 AI 雙手的魔法

Aura
2025-10-16
Temperature 1.5
LangChain, Python

本文為個人學習筆記，記錄了學習過程中的一些知識，可參考，但不可認真，學習的過程可能有理解錯誤，資訊不一定正確，畢竟 Temperature 都 1.5 了。

零、 前言

如果說 Structured Output 是讓模型學會「說官話」（標準格式），那麼 Tools 就是賦予模型「動手做」的能力。

在 LLM 的世界裡，模型本質上只是一個機率預測器，它不知道昨天的天氣，也無法幫你下單買珍珠奶茶。但透過 LangChain 的 Tool 抽象層，我們可以把任何 Python 函數封裝成 LLM 可理解的「技能」，讓 AI 真正從「聊天機器人」進化為「人工智慧代理人 (Agent)」。

一、 什麼是 LangChain Tool？

在 LangChain 的定義中，一個 Tool 包含三個核心要素：

  1. **名稱 (Name)**：告訴模型這個工具叫什麼（例如：get_weather）。
  2. **描述 (Description)**：這是最重要的部分。模型根據這段文字判斷「何時」該用這個工具。
  3. **輸入結構 (Args Schema)**：定義工具需要的參數，通常基於 Pydantic。

當 LLM 發現它無法直接回答問題時，它會輸出一個「工具調用請求」（Tool Call），包含工具名稱與參數。接著，由我們的程式碼執行該工具，並將結果（Observation）回傳給模型，模型再根據結果生成最終回答。

二、 定義工具的三種姿勢

在 LangChain 中，根據複雜程度，我們有不同的方式來定義工具。

1. 最優雅的方式：@tool 裝飾器

這是最常用且推薦的方式，適合大多數簡單場景。LangChain 會自動將函式的 Docstring 視為工具描述。

2. 最嚴謹的方式：繼承 BaseTool

當你需要更複雜的邏輯（例如類別初始化、非同步處理、或者需要維持內部狀態）時，繼承 BaseTool 是最好的選擇。

3. 快速封裝：StructuredTool.from_function

適合將既有的函數快速轉換為工具，而不需要修改函數本身的原始碼。

三、 工具的演進：從 Tool 到 Toolkits

當你的 AI 需要處理複雜任務時，單個工具往往不夠。LangChain 提供了 Toolkits 的概念，將一組相關的工具打包在一起。

Toolkit 類型 包含內容 適用場景
SQL Database 查詢表格、檢查 Schema、執行 SQL 資料庫分析助手
Gmail Toolkit 讀取郵件、草擬、發送、搜尋 自動化辦公助理
Playwright 導覽網頁、點擊按鈕、抓取內容 網頁自動化與爬蟲
Python REPL 執行 Python 代碼 數學計算、數據視覺化

四、 如何讓模型學會用工具？

定義好工具後，我們需要透過 .bind_tools() 將其綁定到支援 Tool Calling 的模型上。

五、 實戰中的「坑」與防禦性編程

雖然 Tool Calling 看起來很神，但在生產環境中，這往往是出問題最多的地方。

1. 幻覺調用 (Hallucinated Tools)

模型有時會發瘋，調用一個你根本沒定義的工具。

  • 對策：使用 StructuredTool 強制校驗參數，並在代碼層級加入 Try-Catch。

2. 安全性：不要讓 AI 擁有無限權力

如果你給了 AI 一個 execute_query 工具，它可能會不小心執行 DROP TABLE users;。

  • 對策：
  • 唯讀權限：給予資料庫唯讀帳號。
  • **人機協作 (Human-in-the-loop)**：在執行敏感動作（如發郵件、扣款）前，增加人工審核環節。

3. 描述 (Description) 競爭

如果你有兩個工具描述得太像（例如 search_google 和 search_bing），模型會陷入糾結。

  • 對策：確保每個工具的 description 有明確的邊界與區隔。

六、 與 LangGraph 的深度聯動

在 LangChain 的新時代，我們不再傾向於使用黑盒子般的 AgentExecutor，而是使用 LangGraph。

在 LangGraph 中，工具調用變成了一個明確的節點（Node）。

  1. **Node 1 (LLM)**：決定要調用哪個工具。
  2. **Edge (Conditional)**：如果 tool_calls 存在，指向 Node 2。
  3. **Node 2 (Tools)**：執行 Python 函數。
  4. **Edge (Loop)**：將結果傳回 Node 1 總結。

這種結構讓整個決策過程變得可視化且可控，是目前開發 AI Agent 的業界標準。

七、 結語

「工具是人類能力的延伸，也是 AI 智能的落腳點。」

掌握了 LangChain Tools，你就握住了通往實用性 AI 的鑰匙。它讓 AI 從「談話型」進化為「行動型」。雖然配置過程中有許多細節需要打磨，但當你看到模型自動查表、計算並發出報告的那一刻，你會發現這一切都是值得的。

別再讓你的 AI 只是「坐著聊天」了，給它一把扳手，讓它動起來吧！
