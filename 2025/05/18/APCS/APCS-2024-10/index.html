1. 裝飲料

解題步驟

  1. 讀取輸入資料: 讀取倒飲料的次數 n，以及杯子兩層的維度 w1, w2, h1, h2。
  2. 計算容量並初始化: 根據維度計算下層的容量 low (即 $w_1 \times w_1 \times h_1$) 以及杯子的總容量 high (即 low + $w_2 \times w_2 \times h_2$)。
  3. 初始化狀態變數: 初始化總裝入容量 total 為 0，前一次的高度 prev 為 0，以及答案 (最大高度變化) ans 為 0。
  4. 模擬倒水過程: 使用 for 迴圈執行 n 次。
  5. 更新總容量與當前高度: 在迴圈中，讀取該次倒入的容量 water，並將其加入 total。
  6. 判斷水位高度:
    • 如果 total <= low，表示水位還在下層，curr 高度為 total / (w1*w1)。
    • 如果 total <= high，表示水位已達上層，curr 高度為 h1 + (total - low) / (w2*w2)。
    • 如果 total > high，表示水已裝滿，curr 高度為 h1 + h2。
  7. 更新最大變化量: 計算本次的高度變化 curr - prev，如果這個值大於 ans，則更新 ans。
  8. 更新前次高度: 將 prev 更新為 curr，以供下次迴圈使用。
  9. 輸出結果: 迴圈結束後，輸出 ans。

分段步驟程式碼

讀取輸入資料

這段程式碼讀取倒飲料的次數 n 和杯子的四個維度參數。

計算容量並初始化變數

這段程式碼計算下層容量 low 和總容量 high。並初始化總容量 total、前次高度 prev 和最大高度差 ans。

模擬 n 次倒水過程並計算高度

這段程式碼是解題的核心。它迴圈 n 次，每次：

  1. 讀取當次倒入的容量 water。
  2. 更新總容量 total。
  3. 根據 total 和 low、high 的關係，計算出當前的水位高度 curr。
  4. 計算本次的高度變化 curr - prev，並更新最大變化量 ans。
  5. 將 prev 設為 curr，為下一次計算做準備。

輸出最大變化量

這段程式碼在所有倒入動作完成後，輸出記錄到的最大高度變化 ans。

2. 蒐集寶石

解題步驟

  1. 讀取輸入: 讀取地圖大小 M、N，倍數 k，以及機器人初始位置 r、c。
  2. 初始化地圖與邊界: 宣告一個比 M x N 稍大的二維陣列 (例如 102 x 102)。讀取 M x N 的地圖資訊，並在地圖的四周圍上一圈牆壁 (-1)。這樣可以簡化後續的邊界判斷。
  3. 初始化機器人狀態:
    • 初始化方向 d = 0 (0:右, 1:下, 2:左, 3:上)。
    • 初始化分數 score = 0。
    • 初始化撿到的寶石數 ans = 0。
    • 將 r 和 c 加 1，以配合我們在地圖周圍加上的牆壁邊界 (變為 1-based 索引)。
  4. 模擬遊戲過程 (主迴圈): 使用 while 迴圈模擬遊戲，迴圈繼續的條件是機器人當前位置 g[r][c] 的寶石數不為 0 (規則 1)。
  5. 執行規則 2 (撿寶石):
    • ans 增加 1。
    • score 加上 g[r][c] 的寶石數。
    • g[r][c] 減 1 (撿走一顆)。
  6. 執行規則 3 (檢查分數): 檢查 score 是否為 k 的倍數。如果是，方向 d 向右轉 90 度 (d = (d + 1) % 4)。
  7. 執行規則 4 (移動與轉向):
    • 使用一個 for 迴圈（最多 4 次）來尋找下一步。
    • 根據當前方向 d 計算出下一步的座標 (new_r, new_c)。
    • 檢查 g[new_r][new_c] 是否為牆壁 (-1)。
    • 如果不是牆壁: 更新機器人位置 r = new_r, c = new_c，並 break 這個 for 迴圈（代表成功移動，準備進入下一次 while 迴圈）。
    • 如果是牆壁: 機器人向右轉 90 度 (d = (d + 1) % 4)，並繼續 for 迴圈，嘗試新的方向。
  8. 輸出結果: 當 while 迴圈結束時 (機器人停在寶石數為 0 的格子)，輸出 ans。

分段步驟程式碼

讀取輸入與初始化邊界

  • (註：原始程式碼中 g[M+1][i+1] 應為 g[M+1][i] 才能正確關閉下邊界，但在此範例中不影響主要邏輯)
  • 這段程式碼讀取基本參數，並在 g[1][1] 到 g[M][N] 的外圍都填上 -1，這樣機器人移動時無需額外判斷是否超出 M x N 範圍，只要檢查是否撞牆 (-1) 即可。

初始化機器人狀態

  • d=0 代表初始方向向右。
  • score 和 ans 均從 0 開始。
  • r++ 和 c++ 將 0-based 索引的輸入轉換為 1-based 索引，以對應我們加了邊界的地圖。

模擬遊戲過程

  • while(g[r][c]!=0): 這是規則 1，只要當前格子寶石數不為 0 就繼續。
  • 規則 2 & 3: 程式碼直接對應題目規則。
  • 規則 4 (for 迴圈): 這是最關鍵的部分。
    • 它首先嘗試當前方向 d。
    • 如果 g[new_r][new_c] 不是 -1 (牆壁)，就更新 r, c 並 break，完成移動。
    • 如果 g[new_r][new_c] 是 -1，就執行 d = (d+1)%4 (向右轉)，for 迴圈繼續，嘗試新的方向。
    • 這個 for 迴圈保證機器人最終會找到一個非牆壁的方向並移動過去。

輸出最終結果

  • 當 while 迴圈終止 (踩到 0)，輸出總共撿到的寶石數 ans。
