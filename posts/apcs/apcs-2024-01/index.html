1. 遊戲選角#

解題步驟#

  1. 讀取角色數量和資料：讀取角色總數 n。使用迴圈讀取 n 次，每次讀取一個角色的攻擊力 a 和防禦力 d。
  2. 儲存資料：使用一個二維 vector (或結構陣列) 來儲存每個角色的攻擊力和防禦力。vector<vector<int>> s 中，s[i][0] 儲存第 i 個角色的攻擊力，s[i][1] 儲存防禦力。
  3. 定義比較函式：要根據「能力值」（$攻擊力^2 + 防禦力^2$）來排序。我們需要自訂一個比較函式 cmp，它接受兩個角色資料 (例如 vector<int> a 和 vector<int> b)。
  4. 實作比較邏輯：在 cmp 函式中，計算 a 的能力值 (a[0]*a[0]+a[1]*a[1]) 和 b 的能力值 (b[0]*b[0]+b[1]*b[1])。為了得到第二大的值，我們需要從大到小 (descending) 排序，所以函式應回傳 a 的能力值是否大於 b 的能力值。
  5. 排序：使用 std::sort 函式，搭配自訂的 cmp 比較函式，對儲存所有角色資料的 vector 進行排序。
  6. 輸出結果：排序後，vector 中的第 0 個元素 ( s[0] ) 具有最大的能力值。因此，第 1 個元素 ( s[1] ) 具有第二大的能力值。輸出 s[1][0] (攻擊力) 和 s[1][1] (防禦力)。

#include<iostream>
#include<vector>
#include<algorithm>

using namespace std;

// 自訂比較函式
// 根據能力值 (a*a + d*d) 由大到小排序
bool cmp(vector<int> a, vector<int> b){
    return a[0]*a[0]+a[1]*a[1] > b[0]*b[0]+b[1]*b[1];
}

這段程式碼包含了必要的函式庫 (iostream, vector, algorithm)。 並定義了一個比較函式 cmp，用於 sort。它計算兩個輸入 vector (分別代表兩個角色) 的能力值平方和，並回傳第一個是否大於第二個，以實現降冪排序。

#include<iostream>
#include<vector>
#include<algorithm>

using namespace std;

// 自訂比較函式
// 比較兩個角色 (a, b) 的能力值
// 能力值 = 攻擊力^2 + 防禦力^2
// 回傳 true 表示 a 應該排在 b 前面 (降冪排序)
bool cmp(vector<int> a, vector<int> b){
    return a[0]*a[0]+a[1]*a[1] > b[0]*b[0]+b[1]*b[1];
}

int main(){
    int n;
    cin >> n ; // 讀取角色數量

    // 宣告 n x 2 的二維 vector 儲存資料
    vector<vector<int>> s(n,vector<int>(2));

    // 迴圈讀取 n 個角色的攻擊力 (s[i][0]) 和防禦力 (s[i][1])
    for(int i=0;i<n;i++){
        cin >> s[i][0] >> s[i][1];
    }

    // 使用 std::sort 和自訂的 cmp 函式對角色
    // 根據能力值進行降冪排序
    sort(s.begin(),s.end(),cmp);

    // 排序後，s[0] 是第一名，s[1] 是第二名
    // 輸出第二名角色的攻擊力和防禦力
    cout << s[1][0] << ' ' << s[1][1];
}

2. 蜜蜂觀察#

解題步驟#

  1. 讀取輸入: 讀取蜂巢大小 $m$、$n$ 以及總步數 $k$。
  2. 儲存蜂巢地圖: 建立一個 $m \times n$ 的二維資料結構（例如 vector<string>），並讀取 $m$ 行字串來儲存蜂巢上每個格子的字母。
  3. 初始化狀態:
    • 設定 Bob 的起始位置 (x, y) 在左下角，即 (m-1, 0)。
    • 建立一個大小為 52 的整數陣列 abc 並全部初始化為 0，用來分別統計 26 個大寫字母和 26 個小寫字母出現的次數。
  4. 模擬移動過程:
    • 使用一個 for 迴圈執行 $k$ 次，代表 Bob 的 $k$ 步移動。
    • 在迴圈中，首先讀取該步驟的移動方向 step。
    • 檢查邊界並移動: 根據 step 的值 (0-5)，判斷移動後的 新位置是否會超出蜂巢邊界（撞牆）。
      • 如果不會超出邊界，則更新 x 和 y 座標。
      • 如果會超出邊界，x 和 y 座標保持不變（停在原地）。
    • 記錄路徑與字元:
      • 輸出當前位置 (x, y) 上的字元 comb[x][y]。
      • 將 abc 陣列中對應的字元計數加 1。需區分大小寫，例如 ‘A’-‘Z’ 對應 0-25，‘a’-‘z’ 對應 26-51。
  5. 計算並輸出字元種類數:
    • 迴圈結束後，宣告一個變數 diff 來計算種類數。
    • 遍歷 abc 陣列 (0 到 51)，如果 abc[i] > 0（表示該字元至少出現過一次），則將 diff 加 1。
    • 最後，換行並輸出 diff 的值。

    }

這段程式碼是 k 步移動的主迴圈。

  1. 讀取每一步的方向 step。
  2. 使用 if-else if 結構判斷 6 種方向。
  3. 在更新座標 x 和 y 之前，先檢查新位置是否仍在 $m \times n$ 的範圍內。如果超出範圍（撞牆），則不更新座標，Bob 停在原地。

  1. cout << comb[x][y]; 輸出當前（移動後或停在原地）位置的字元，累加成路徑字串。
  2. if-else 判斷當前字元是小寫還大寫：
    • 大寫：‘A’ 對應 abc[0]，‘B’ 對應 abc[1] …
    • 小寫：‘a’ 對應 abc[26]，‘b’ 對應 abc[27] … 並將對應的計數器加 1。

int diff = 0;

cout<<endl<<diff;

這段程式碼在 for 迴圈結束後執行。

  1. 初始化種類數 diff 為 0。
  2. 遍歷 abc 陣列，abc[i] > 0 這個布林表達式在 C++ 中會被轉為 1 (true) 或 0 (false)。
  3. 因此，diff 只會累加那些至少出現過一次 (計數 > 0) 的字元種類。
  4. 最後，cout << endl; 換行，並輸出總種類數 diff。
